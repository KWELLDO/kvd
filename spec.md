# KVD 文件格式规范 v1.0

## 概述

KVD (KWD Versioned Document) 是一种自带版本历史的纯文本文件格式。
每个 `.kvd` 文件本身就是一个小型版本仓库，记录了文件内容的每一次变更——
改了什么、什么时间、谁改的，一目了然。

## 设计目标

- **自包含** — 无需 `.git` 目录或外部依赖，一个文件就是一个仓库
- **人类可读** — 用记事本、`cat`、`less` 打开即可看到完整历史和当前内容
- **机器可解析** — 结构清晰，工具可以精确读写
- **可验证** — 每个提交带哈希链，内容篡改可被检测
- **极简** — 格式简单到可以手写

## 文件扩展名

`.kvd`

## 格式结构

一个 KVD 文件由三部分组成：

```
KVD/v1                                 ← 文件头
Title: <标题>
Created: <创建时间>

--CURRENT--                            ← 当前内容区
<文件的当前内容>

--COMMITS--                            ← 历史提交区
>>> <序号> | <哈希>
author:<作者>
date:<时间>
msg:<提交信息>
parent:<父提交序号>
---
<该提交时的完整内容>
<<< <序号>
```

### 1. 文件头

第一行必须是 `KVD/v1`。
`Title` 可选，默认为文件名。
`Created` 可选，自动生成。

### 2. 当前内容区

以 `--CURRENT--` 开头，后面是文件的最新内容。
内容可以是任意文本，包含中文、空行等。

### 3. 历史提交区

以 `--COMMITS--` 开头，后面按时间从旧到新排列提交记录。

每条提交记录格式：

```
>>> <序号> | <哈希>
author:<作者>
date:<ISO 8601 时间>
msg:<提交信息>
parent:<父提交序号>
---
<内容>
<<< <序号>
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| 序号 | 是 | 从 1 递增的整数 |
| 哈希 | 是 | SHA256 截断前 8 位 |
| author | 是 | 提交者名称 |
| date | 是 | ISO 8601 格式时间 |
| msg | 是 | 提交信息 |
| parent | 否 | 父提交序号 |
| --- | 是 | 元数据与内容分隔线 |
| 内容 | 是 | 该提交时的完整文件内容 |
| <<< N | 是 | 提交结束标记 |

### 哈希计算

每个提交的哈希 = SHA256(parent_hash + content + author + date + message) 的前 8 位

- 首个提交的 parent_hash 为空字符串
- 哈希链保证了历史记录的完整性

## 取值约束

- 所有文本行不应以 `>>>`、`<<<`、`--CURRENT--`、`--COMMITS--` 开头
- 编码统一使用 UTF-8 (无 BOM)
- 换行符统一使用 LF (`\n`)

## 使用场景

- **配置文件** — 需要审计追踪，谁改了哪个参数一目了然
- **工作日志** — 每天记日志自动附带版本历史
- **笔记文档** — 不需要完整 git 仓库，但又想保留修改痕迹
- **共享文件** — 通过网盘或邮件传输时，对方直接看到完整演变
- **合规审计** — 文件级别的变更追踪，满足合规要求
